初始化本地仓库
	git init   会创建一个.git 文件夹

从在线仓库上去拉取代码
	git clone 地址   --------------- 首次拉取代码,只拿到了代码的一个备份,并没有仓库分支/指针
	git pull 地址 	---------- 从在线仓库上合并下来代码,这时才能用仓库操作,否则容易报错

本地写了一些代码 , 创建加了一些文件 需要让git追踪我们的文件
	检查当前仓库中有无文件变动
		git status
	暂存区
		git add *
	提交本地仓库并记录本次提交
		git commit -m '本次记录'

	--------------------假如没有配置在线仓库的秘钥
			创建本地的ssh秘钥	ssh-keygen  -t rsa
			把我的电脑/users/当前用户/.ssh文件夹/id_rsa.pub 文件内容拷贝到仓库秘钥中
				(配置秘钥步骤:
					1 打开github 账户
				)
			配置完成就可以去提交了

提交到在线仓库
	准备好在线仓库地址? github上找仓库地址 复制
	--------------------假如 以后会长期操作当前这个在线仓库,我们需要给这个仓库地址配置一个容易记住的别名 origin  abc  banana...
			git remote add 别名 仓库地址		配置完成后就可以使用这个别名操作在线仓库了
	git push -u 别名/地址 分支名			git push -u origin master首次提交一定要加 -u


分支操作:
	创建分支  
		git branch 分支名
	切换分支
		git checkout 分支名
	合并分支
		在某一个分支上执行合并	git merge 分支名  -----------  会把分支名上的代码 合并到 当前分支上
